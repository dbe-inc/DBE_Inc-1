      4. Third Priority: Folder Structure
        1. All work will be uploaded to GitHub, which is managed by the Computer Dept.

    2. Assigning Duties: Who does What.
      1. Counter Module
        1. Program Counter
        2. Micro Counter (Could be included in the Control Module)
        3. Multiplexer for the four 7-Segment Displays
          (Cound be included in the Output Module)
      2. Arithmetic Logic Unit (ALU) Module
        1. Arithmetic Unit (AU)
        2. Logic Unit (LU) Only if extra time is available!
        3. Error Flag(FC) for both + and +/- numbers(2S)
        4. Sum = 0 Flag(FZ)
        5. Flag Register to store FC and FZ flags.
      3. Output Module
        1. 8-bit Register
        2. Multiplexer Clock plus demultiplexer
        3. Four(4) 7-Segment Displays.
        4. One EEPROM. Like the two Control EEPROMs, this EEPROM will also be programmed by the Computer Dept. This requires no input from the Engineering Dept but be sure to co
        5. One Control Signal(2S) which will connect to address a10 of the EEPROM. When 2S is connected to ground(GND) the EEPROM will output only positive numbers [0,255].
          6. When 2S is connected to 5V(VCC) the EEPROM will ouput +/- numbers [-128,127].
      4. Clock Module
        1. Clock (LM555)
        2. Push Botton for Step by Step operation
        3. Switch which will allow for either 'Continous'(Clock) or 'Step by Step'(Button) operation.
        4. Provide 2nd Clock Signal is out of sync with Clock. eg when clock is '0' 2nd clock is '1' and vice versa. Hint: NOT gate!
      5. BUS Module
        1. Possible Option(Engineering Manager's responsibility
        2. Design the BUS which MUST include the 74LS245 Transceivers.
          Although Ben Eater places all Transceivers on the individual Modules, the Transceivers provide the integrity of the BUS and, therefore, should remain with the BUS Designer!
        3. Add Modules to the BUS as these Modules are completed.
      6. Memory(RAM) Module:
        1. Memory Address Register(4-bit)
        2. Manual Address Input (4-bit DIP Switches)
        3. 4 bit Multiplexer between Memory & Manual Addresses
        4. Run/Program Switch.
        5. 16 Byte RAM
        6. Manual Data Input (8-bit DIP Switches
        7. 8 bit Multiplexer between Memory & Manual Data
      7. Control Module
        1. Instruction Register(8-bit)
        2. Micro Step Counter with decoder
        3. 2 EEPROMs: NOTE: The programming of these EEPROMs will be done by the Computer Department.
          This will require coordination between the Engineering & Computer Depts. The Engineering Dept must first allocate each Control Signals to the EEPROM outputs after which the Computer Dept can actually program them!
        4. While Ben Eater has ALL Instructions decoded in the EEPROMs, for this project, ALL Jump instructions MUST be handled with "discrete components".
          eg AND, OR, NOT gates (74LS08, 74LS32, 74LS04 ICs)
    3. Quality Control
      1. "Digital" has the capability to run a test on each design.
      2. While the Engineer should run the tests, The Engineering Manager should probably be the one to set up the Tests themselves.
        1. Example:Tests for Program Counter (74LS161)
          1. Test: counting from 0 to 15
          2. Test: input 4-bit data
          3. Test: ability to Clear count
        2. Example:Tests for ALU
          NOTE: This will be challenging to test for every possibility.
          1. Test: for accuracy of addition of some! #'s as well as correct values for flags FC & FZ
          2. Test: for accuracy of subtraction of some! #'s as well as correct values for flags FC & FZ.
          3. NOTE: for adding two numbers there are 256x256 posibilities (65,536)
          4. Note: Testing in "Digital" does provide the capabilities to loop! to handle problems like above.
